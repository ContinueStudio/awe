## v0.3.7k — 流程编辑器顶栏可拖动
- **Bug**: 主界面顶部栏能拖，但进入流程编辑后顶栏点不动
- **根因**: `App.tsx` 里 editor 模式（`view.kind === 'editor'`）走的是全屏布局，没有渲染 `CustomTitleBar`；编辑器自己的 `<header>` 没有绑定任何拖动 handler
- **修复**:
  - 抽出 `useWindowDrag()` hook，统一维护拖动状态（`start_drag` + `move_window_delta`）
  - `CustomTitleBar` 改为接收 `onMouseDown` prop，与 App 共用同一份 handler
  - 编辑器 `<header>` 加 `onMouseDown={onWindowDragMouseDown}`，并给内部按钮/输入框加 `onMouseDown={stopDrag}` 阻止误触发拖动
- **验证**: `npm run build` 通过，dist 已更新
- **启动**: `start.bat`（窗口模式）直接生效
- **Docs**: PRD v2.30 → v2.31，DEV v0.3.7j → v0.3.7k

- **Bug 1**: 桌面窗口模式点击顶部栏无法拖动窗口（主界面和节点编辑界面都不行）
  - **根因**: `frontend/src/App.tsx` 里 `mousemove` 调的是 `api.move_window(newX, newY)`，但 `desktop/launch_window.py` 只暴露了 `move_window_delta(dx, dy)`，函数名和参数双重不匹配，拖动指令根本没发到后端
  - **修复**: `App.tsx` 改发鼠标增量并调 `move_window_delta(dx, dy)`
- **Bug 2**: 能拖动后，窗口越往右下角点击标题栏越往右下角跳
  - **根因**: `launch_window.py` 里 `start_drag` 用 `GetWindowRect` 拿到的是**物理像素**，但 `window.move()` 接收的是**逻辑像素**（pywebview 内部会再乘以缩放），两者混用导致高 DPI/缩放时越远离原点偏移越大
  - **修复**: `launch_window.py` 里 `_win_x/_win_y` 全程按逻辑像素维护；`start_drag` 把 `GetWindowRect` 结果除以 `window.native._scale` 转成逻辑像素；`move_window_delta` 保持逻辑增量相加
- **验证**: `npm run build` 通过，`py_compile` 通过
- **启动**: `start.bat`（窗口模式）直接生效，无需重新构建前端
- **Docs**: PRD v2.29 → v2.30，DEV v0.3.7i → v0.3.7j

## v0.3.7i — WebView2 拖动用 pywebview DRAG_REGION_SELECTOR
- **Bug**: v2.28 用 `.awe-titlebar { -webkit-app-region: drag }` 标题栏**仍然不能拖**
- **根因**: **WebView2 不支持 `-webkit-app-region`**（只对 Electron/NW.js 生效）
- **修复**:
  - `launch_window.py` 加 `webview.DRAG_REGION_SELECTOR = ".awe-titlebar"`
  - pywebview customize.js 在 mousedown 时调 pywebviewMoveWindow API
  - 走 Python 端处理，绕开浏览器层限制
- **避坑**:
  - WebView2 不支持 `-webkit-app-region`
  - pywebview frameless 拖动靠 Python 端（customize.js）
  - 不要混用两套机制
- **Docs**: PRD v2.28 → v2.29，DEV v0.3.7h → v0.3.7i
- **GitHub**: 已推送 commit af08ef7
