# 2026-07-06

## v0.3.8: 复制粘贴 + 路径参数 + 自定义确认对话框 (PRD v2.30)

### 1. 修复节点复制按钮无反应
- App.tsx: `duplicateNode(id)` 函数，通过 Canvas → NodeRender `onDuplicate` 传递
- 点击节点内复制按钮：克隆节点（新 id、offset +50/+50），同时存入剪贴板

### 2. 粘贴功能和 Ctrl+C/V 快捷键
- App.tsx: `clipboard` 状态（CanvasNode[] | null），支持跨工作流粘贴
- `copySelectedNodes()`: Ctrl+C 复制选中节点
- `pasteNodes()`: Ctrl+V 粘贴，新 id + 动态偏移
- 按键不拦截输入框（INPUT/TEXTAREA/isContentEditable）
- Canvas.tsx: 画布右键菜单、`onDuplicateNode` 和 `onPaste` props

### 3. 读写节点路径参数展示
- knowledge: `storage_path` → ./data/chroma
- sqlite: `db_path` → ./data/awe.db  
- excel_save_records: `filename` → `file_path` → ./data/output.xlsx
- skill: `working_dir` → ./data/skills
- builder.py: 各函数优先用 cfg 中的路径参数
- sandbox.py: working_dir 注入为全局变量

### 4. 自定义确认对话框
- ConfirmDialog: 白底+slate 边框+fixed 居中+无提示音+中文
- HomePage: 替换 confirm()/alert()
- Python: confirm_close=False + force_close + on_closing

### 5. 删除按钮修复 + 复制只存剪贴板 + 运行前自动保存 + 错误节点高亮 (patch 2)
- **删除按钮**: Canvas.tsx 添加 `onDeleteNode` prop 并传递给 NodeRender
- **复制按钮**: `duplicateNode` 不再立即创建节点，仅存入剪贴板并提示 "Ctrl+V 粘贴"
- **运行前自动保存**: `handleRun` / `handleRunSingleNode` / `handleRunSelectedNodes` 都先调用 `handleSave()`，解决填了 `file_path` 但提示缺少的问题
- **错误节点高亮**: 
  - 新增 `errorNodeIds` state，运行失败时解析错误信息中的节点 ID 并高亮红色边框
  - 运行成功时也从 `result.logs` 中提取失败的节点 ID
  - NodeRender 新增 `error` prop，运行时出错的节点显示红色边框 + 红色阴影
  - 选中节点时自动清除该节点的错误高亮
- **左侧运行日志**: 每次运行都自动弹出 `RunHistoryDrawer`（`setShowRunHistory(true)`）
- **去掉 alert**: 所有运行失败改用 `setToast` + 显示日志面板，不再弹出浏览器 alert
- 涉及文件: App.tsx, Canvas.tsx, NodeRender.tsx
