# AWE 项目长期记忆

## 铁律
- **PRD 同步硬规则**：任何涉及功能/UI/状态机/字段的代码变更，必须同步更新 PRD 文档
- **DEV 同步硬规则**：每次改代码必须同步更新 DEV 文档（改动+避坑+验证步骤）
- **自启自检**：每次修改后必须先 kill 旧进程 → 重新构建 → 启动 → 健康检查 → 否则不汇报
- **GitHub 备份**：自检通过后立即 commit + push
